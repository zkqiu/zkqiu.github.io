Published on February 18, 2026 | 5 min read

Recently, I set up OpenClaw ‚Äî a powerful AI agent framework that runs locally and integrates with my daily tools. Here's my complete setup journey, from installation to configuring email monitoring and automated reminders.

üí° What is OpenClaw? OpenClaw is a local AI agent system that can control your browser, manage files, send messages, schedule tasks, and much more ‚Äî all while keeping your data private on your own machine.

1Installation

OpenClaw is installed via npm. The installation is straightforward:

npm install -g openclaw

After installation, you can verify it's working:

openclaw --version

2Initial Configuration

The first run requires some basic setup. OpenClaw creates a workspace directory at ~/.openclaw/workspace with essential configuration files:

  • SOUL.md ‚Äî Defines the AI's personality and behavior
  • IDENTITY.md ‚Äî Your AI's name and characteristics
  • USER.md ‚Äî Information about you (the user)
  • AGENTS.md ‚Äî Operating guidelines and best practices
  • memory/ ‚Äî Directory for session memories and notes

3WhatsApp Integration

One of the first things I configured was WhatsApp messaging. OpenClaw can send and receive messages through WhatsApp, making it easy to interact with the AI from my phone.

The setup involves scanning a QR code to link your WhatsApp account. Once connected, you can chat with your AI agent just like any other contact!

4Gmail Monitoring Setup

This was the most interesting part. I wanted OpenClaw to automatically check my Gmail and notify me of important messages.

Option A: OAuth (Google Cloud Console)

Initially, I tried the OAuth route, which requires:

  • Creating a project in Google Cloud Console
  • Enabling Gmail API
  • Creating OAuth credentials
  • Adding test users (since the app isn't verified)
‚ö†Ô∏è Challenge: The OAuth app shows "not verified" warnings for new projects. You need to add your email as a test user in the consent screen settings.

Option B: IMAP (Simpler!)

I ended up using the simpler IMAP approach with an App Password:

  1. Enable 2FA on your Google Account
  2. Generate an App Password at myaccount.google.com/apppasswords
  3. Configure IMAP settings in OpenClaw
# IMAP Configuration
IMAP Server: imap.gmail.com
Port: 993 (SSL)
Username: your.email@gmail.com
Password: [App Password]

5Scheduling Automated Checks

With Gmail connected, I set up a cron job to check for new emails every 30 minutes:

# Runs every 30 minutes
*/30 * * * * openclaw check-email

This ensures I never miss important messages, and OpenClaw can even help draft replies!

6Memory & Context

OpenClaw maintains memory across sessions using markdown files:

  • memory/YYYY-MM-DD.md ‚Äî Daily session logs
  • MEMORY.md ‚Äî Long-term curated memories

This allows the AI to remember context from previous conversations and build a better understanding of your needs over time.

üîß Tips & Lessons Learned

  • Start simple: Don't try to configure everything at once. Get the basics working first.
  • IMAP > OAuth: For Gmail, the IMAP approach with App Password is much simpler than OAuth.
  • Memory matters: Take time to review and curate your MEMORY.md file periodically.
  • Security first: Never commit sensitive credentials to git. Use environment variables.
